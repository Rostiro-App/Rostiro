Rostiro — Onboarding Sequence & Mode System

July 2, 2025 · Design reference for Claude Code
Onboarding sequence — 6 steps
1
Mode selection — before account creation
"How do you run your leagues?" Three cards — Focused, Balanced, Savant. User picks one. This is the very first interaction — before email, before password. Sets the mental model immediately: you're configuring an OS, not signing up for a tool.
Mode is stored in localStorage until account is created, then persisted to the users table.
Mode selectionNo auth yetSets identity
2
Create account
Email + password or magic link. Copy: "Your Rostiro OS is ready. Create your account to save it." Framed as completing a configuration — not starting a signup. 7-day full Starter trial begins automatically on confirm.
Never say "Sign up." Say "Save your setup" or "Activate your OS."
Email / magic link7-day trial starts
3
Connect your leagues
Connect ESPN (cookie guide), Yahoo (OAuth), Sleeper (username). Skip is available but copy is honest: "Connect later — but Rostiro can't help until you do." At least one league required to reach Pulse. ESPN and Yahoo show "Coming soon — join early access" until integrations are approved.
Sleeper works immediately. Build the full flow with Sleeper first. ESPN and Yahoo plug in when ready.
Sleeper live nowYahoo pendingESPN in progress
4
League configuration — scoring variables
For each connected league: scoring format, roster slots, league size, waiver system, FAAB budget, playoff weeks, trade deadline. This is what makes Rostiro's AI league-aware instead of generic. Show as a per-league card — not a long form. Yahoo and Sleeper auto-fetch these values pre-filled. ESPN requires manual input until auth is live.
Never make a variable required if it can be reasonably inferred. Default values shown and editable.
Auto-fetched for Yahoo/SleeperManual fallback always available
5
Enable push notifications — the retention moment
Show the actual notification example first: "Joe Mixon is OUT. Affects 2 leagues. Tap to see moves." Then ask permission. THEN trigger the browser prompt. Pre-framing dramatically improves opt-in rate. iOS users get an explanation of Add to Home Screen. Denied users get email fallback via Resend.
This is the most important step for season-long retention. Don't skip it. Don't bury it.
OneSignal web pushiOS: Add to Home ScreenEmail fallback if denied
6
Your OS is live — land on Pulse
Animated sync screen: "Syncing your leagues... analyzing rosters... building your first Pulse." Then lands directly on the Pulse dashboard — never a settings page or empty state. The first thing the user sees is the north star experience.
If Pulse has nothing to show yet (offseason, no active week), show a "Draft Kit ready" card pointing to rostiro.com/draft as the first action.
Lands on PulseFirst sync complete7-day trial active
The three modes — full definitions
⚡
Focused
"Tell me what to do."
Rostiro makes the decisions. User executes. Everything distilled to the minimum actions needed to win this week. No noise, no data overload. In and out in under 3 minutes.
5 prioritized actions maximum per Pulse session
Verdict shown before reasoning — decision first, data second
One-tap actions — no intermediate confirmation screens
Stats hidden by default — tap "why" to see reasoning
Estimated session time shown on every Pulse view
Push notifications on by default
The manager in 5 leagues who wants to win without spending hours on Sunday morning.
⚖️
Balanced
"Show me the key stuff."
The middle ground. Rostiro surfaces decisions with the most relevant data behind each one. User sees enough to feel in control without being overwhelmed by a full data layer.
Actions shown with key supporting data inline
Matchup difficulty, injury status, weather always visible
Tap any item to expand to full Savant data view
Cross-league exposure shown on roster view
Trade and waiver reasoning visible by default
ADP delta and scarcity shown on draft board
The manager who trusts data but still wants to make the final call with context in hand.
🧠
Savant
"Give me everything."
Full portfolio intelligence. Every data point visible. Nothing hidden. Rostiro is the infrastructure — the manager is the analyst. Bloomberg terminal mode.
Full data layer always visible — no collapsed stats
Cross-league exposure bars on every player
Vegas totals, target share, snap count, usage trends
Raw projection numbers and confidence intervals
AI recommendations shown but never forced — advisory only
Opponent tendency modeling visible
The manager who already knows what they want — they want full context and confirmation, not direction.
Mode is always changeable — this is critical
Mode lives in user settings and toggles from the sidebar at any time, including mid-session. Focused users who have a tough decision week will flip to Balanced to dig in. Savant users on a busy weekend will flip to Focused to get through it fast. The toggle earns trust both ways. Never lock anyone in.
